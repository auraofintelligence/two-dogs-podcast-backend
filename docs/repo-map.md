# Repo Map

## Planning Flow

1. Open `builders/index.html`.
2. Choose the feedback form type.
3. Fill the form.
4. Export the generated `.md` file.
5. Save it into the matching folder.
6. Use `source-references/` when deeper context is needed.
7. Use `handoffs/` when another agent or future session needs a small task packet.

## Naming Pattern

Use simple slugs:

- `episode-2026-05-17-island-country-seed.md`
- `scene-2026-05-17-early-boat-from-cleveland.md`
- `guest-firstname-nickname.md`
- `segment-old-dogs-young-dogs.md`
- `ad-local-supporter-seed.md`

## Keep It Small

Each note should be useful on its own.

If a generated file starts to become too large, split it into a scene, source reference or handoff through the builder. Do not build a complex management system unless the show actually needs it.
