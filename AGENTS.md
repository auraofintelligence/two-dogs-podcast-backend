# Agent Instructions

Use Australian English.

This repo is a lightweight public planning bench for Two Dogs Podcast, not a heavy decision system.

Primary workflow: use the interactive feedback forms in `builders/index.html` to generate `.md` files. Do not hand-write episode, scene, guest, ad, segment, source or handoff Markdown files unless Luke explicitly asks for that direct edit.

## Core Boundaries

- Luke is Red Dog / Red Heeler.
- Angel is Blue Dog / Blue Heeler.
- Do not write prompts, scripts or direction for Blue Dog's voice. Leave Blue Dog space for Angel.
- Do not assign a spirit animal or nickname to a guest. Guests choose their own.
- Red Dog can reference deeper datasets such as Strange But True, Aura, P4A, personal AI, agent systems and future website directions.
- Keep source-heavy or personal notes separate from public-facing episode copy.

## Working Style

Use short, practical form outputs.

Prefer:

- episode notes
- scene notes
- guest notes
- source reference notes
- simple handoffs

Avoid:

- heavy governance layers
- overbuilt decision trees
- fake certainty
- assigning identities to guests
- writing Angel's Blue Dog voice

## Source Use

Before reading deeper source documents, use Luke's local source-navigation layer if it is available in the working environment.

Do not scan the whole source archive. Use the navigation layer to choose one or two genuinely useful source docs, and do not publish raw local file paths in generated public outputs.

## Output Pattern

When generating a new planning file through the builder, include:

- status
- purpose
- Red Dog angle
- Blue Dog boundary
- guest choice boundary if relevant
- source references if used
- next useful action
